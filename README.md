# Wedding Invitation Landing

Лендинг-приглашение на свадьбу Рамиса и Асемы.

Проект сделан как статическая страница на `HTML + CSS + JavaScript` без сборщика. Основной сценарий:
- стартовый экран с закрытым конвертом и восковой печатью;
- музыкальная кнопка;
- свадебный лендинг с таймером, программой дня, деталями и контактами;
- RSVP-форма с отправкой в Google Sheets через Google Apps Script.

## Текущая точка входа

Сейчас в работе участвуют именно эти файлы:
- `index.html` — вся разметка страницы;
- `styles.css` — все стили и адаптив;
- `app.js` — логика интро, таймера, формы, музыки и отправки RSVP;
- `serve.ps1` — локальный сервер для разработки;
- `google-apps-script.gs` — серверная часть для записи ответов в Google Sheets.

Папка `scripts/` в текущей версии страницы не подключена и не является основной точкой входа.

## Как запустить локально

Из корня проекта:

```powershell
powershell -ExecutionPolicy Bypass -File .\serve.ps1
```

После этого сайт будет доступен по адресу:

```text
http://localhost:5500
```

### Открыть с телефона в одной Wi-Fi сети

```powershell
powershell -ExecutionPolicy Bypass -File .\serve.ps1 -Lan
```

Скрипт покажет локальный IP, например:

```text
LAN access: http://192.168.0.106:5500
```

Этот адрес можно открыть на телефоне, если компьютер и телефон находятся в одной сети.

## Как устроена форма RSVP

Форма находится в `index.html`, логика отправки — в `app.js`.

Что происходит:
- данные собираются из формы;
- сохраняются локально в браузере как черновик;
- отправляются в Google Sheets через опубликованный Apps Script Web App;
- при ошибке отправки ответ остаётся сохранённым локально.

Ключевые детали:
- URL Apps Script сейчас хранится прямо в `app.js` в константе `SCRIPT_URL`;
- серверная часть для таблицы лежит в `google-apps-script.gs`;
- таблица заполняется в лист `RSVP`.

## Как подключить Google Sheets

1. Откройте нужную Google Sheets.
2. Перейдите в `Extensions` -> `Apps Script`.
3. Вставьте код из `google-apps-script.gs`.
4. Сохраните проект.
5. Опубликуйте `Deploy` -> `New deployment` -> `Web app`.
6. Дайте доступ `Anyone`.
7. Скопируйте URL вида `/exec`.
8. Вставьте этот URL в `app.js` в константу `SCRIPT_URL`.

Важно:
- после изменения Apps Script недостаточно просто сохранить код — нужно заново обновить deployment;
- если форма не пишет в таблицу, сначала проверьте `Executions` в Apps Script.

## Что чаще всего редактируется

Если нужно быстро обновить контент, чаще всего правятся:
- `index.html` — тексты, ссылки, структура блоков;
- `styles.css` — внешний вид, отступы, адаптив, анимации;
- `app.js` — интро-конверт, таймер, поведение формы, музыка, отправка данных;
- `assets/images/*` — фото пары, конверт, печать, фото площадки;
- `assets/icons/*` — иконки программы, контактов и декоративных элементов.

## Особенности проекта

- Лендинг в первую очередь адаптирован под мобильный формат, но desktop тоже поддерживается.
- Закрытый конверт на старте реализован как fullscreen overlay.
- В проекте есть музыка, которая запускается только по действию пользователя.
- Локальный сервер специально отдаёт файлы с `no-store`, чтобы кэш не мешал при правках.

## Если что-то “сломалось”

Быстрый чек-лист:
- обновить страницу через `Ctrl+F5`;
- на телефоне заново открыть ссылку, а не просто вернуться в старую вкладку;
- убедиться, что локальный сервер запущен;
- проверить, что нужный asset действительно лежит в `assets/...`;
- если не работает отправка формы, отдельно проверить Apps Script и Google Sheets.

## Идея по дальнейшей поддержке

Если проект продолжит жить и меняться, лучше держаться простой схемы:
- разметка — в `index.html`;
- стили — в `styles.css`;
- интерактив — в `app.js`;
- все медиа — только в `assets/` по тематическим папкам.

Так в проекте легче ориентироваться и вносить точечные правки без побочных поломок.
